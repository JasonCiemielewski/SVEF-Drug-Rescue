The Agentic Enrichment Prompt
Create and save a plan that performs the following, allow me to review the plan before you implement:

Role: You are a Senior Bioinformatics Data Engineer. Your goal is to enrich a foundational dataset of efficacy-failed clinical trials with quantitative clinical volume, temporal data, and chemical descriptors.

Objective: Enhance the existing SVEF_candidates dataframe by joining additional AACT relational tables and performing asynchronous lookups on the PubChem PUG-REST API to retrieve molecular signatures.

Context: This is the "Feature Engineering" phase of an IP Rescue project. We are moving from a simple list of names to a data-rich table that quantifies safety exposure and provides the SMILES strings required for Drug-Target Interaction (DTI) modeling.

Task Instructions:

Step 1: Clinical Volume & Duration (AACT Tables)

    Load studies.txt (for enrollment, start_date, completion_date).

    Load calculated_values.txt (for actual_duration).

    Load sponsors.txt (for agency_class – to identify Industry vs. Lead Academic).

    Load browse_conditions.txt (to get mesh_term for the original failed indication).

    Merge Logic: Perform left-joins on the existing dataframe using nct_id. Ensure no row duplication occurs during the join with browse_conditions (collapse multiple MeSH terms into a single pipe-separated string if necessary).

Step 2: Temporal Feature Engineering

    If actual_duration is missing, calculate trial_duration_days as the delta between start_date and primary_completion_date.

    Handle date parsing errors gracefully.

Step 3: Cheminformatics Enrichment (PubChem PUG-REST API)

    Create a function get_pubchem_data(drug_name) that:

        Queries the PubChem API to find the CID (Compound ID).

        Retrieves SMILES, MolecularWeight, and XLogP.

    Constraint: Implement a rate-limiter (e.g., time.sleep(0.2)) to respect PubChem’s request limits.

    Defensive Coding: Use a try-except block to handle cases where a drug name is a combination therapy or not found.

Step 4: Data Quality & Normalization

    Create a Safety_Score column: A normalized value (0.0 to 1.0) based on a weighted combination of enrollment and actual_duration.

    Flag rows that have a valid SMILES string, as these are "DTI-Ready."

Step 5: Final Output

    Export the enriched dataset to data/SVEF_Enriched_Final.csv.

    Generate a summary report:

        Average enrollment of the SVEF candidates.

        Number of candidates successfully matched to a SMILES string.

        Distribution of "Industry" vs. "Academic" assets.

Technical Constraints:

    Use Python 3.12+ in the established .svef virtual environment.

    Use pandas for data manipulation and requests for API calls.

    Pathing must be relative to the project root: ./data_raw/ for AACT and ./output/ for results.
